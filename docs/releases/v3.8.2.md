# chb v3.8.2

Release date: 2026-07-01

A patch release hardening the Wise → Odoo sync against re-duplication, sourcing
KBC balances from Odoo when the journal is authoritative, and adding a surgical
GL reclassify command.

## Highlights

### Wise
- **`chb wise sync` is now report-only for a closed journal.** A journal that
  already carries a year-end closing plug (a balance entry dated after the last
  CSV transaction) is treated as closed/reconciled: `--apply` creates nothing
  and the opening/closing writes are skipped. Such journals were imported at a
  different granularity than the raw Wise CSV — fees folded into the gross
  payment, off-by-one dates — so the leftover "missing" rows are artefacts, not
  real gaps, and re-creating them would duplicate. Pass `--force` to override.
  An open/fresh journal (no post-range plug) still syncs normally.
- **Stable import ids for round-trips.** When a single Wise `TransferID`
  survives as two opposite-sign legs (a transfer sent then returned), the extra
  leg now carries an `ImportSuffix` so its `unique_import_id` stays unique
  instead of colliding on the Odoo side.

### Accounts
- **KBC balance from Odoo when source-of-truth.** For an `odooSourceOfTruth`
  KBC account the Odoo journal is authoritative, so the live balance is the sum
  of the pulled journal lines (the bootstrap CSV is ignored) and the balance
  source reads "Odoo journal". On pull, the journal lines are materialised into
  the monthly `transactions.json` so `generate`'s aggregate and the local
  balance pick them up like every other account.

### Odoo
- **`chb odoo reclassify <lineId...> --to <accountCode> [--apply]`** moves one
  or more bank-statement lines' counterpart GL account onto a chosen code,
  reusing the same draft → write → post engine as the mapping reclassifier so
  posted/reconciled lines are handled safely. It's the surgical counterpart to
  the bulk `chb odoo reconcile --remap` for one-off mis-tagged lines.

## Upgrade Notes

1. `chb wise sync --apply` is now a safe no-op on the closed Wise journals — it
   no longer re-creates duplicates. To force an import into a closed journal
   (rarely wanted), pass `--force`.
