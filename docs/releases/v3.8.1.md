# chb v3.8.1

Release date: 2026-07-01

A follow-up to v3.8.0 focused on Odoo-source-of-truth accounts, Monerium
enrichment on pull, scoped journal fixes, and an events.json staleness fix.

## Highlights

### Accounts
- **KBC pulls from Odoo, not the CSV.** For an `odooSourceOfTruth` account the
  Odoo journal is authoritative, so `chb accounts kbc pull` now refreshes the
  journal-lines cache from Odoo (and says "from Odoo"), instead of reading the
  bootstrap CSV. The stale CSV-derived "Local" row is dropped from the summary.
- **`chb accounts push`** (no slug) pushes every pushable account, skipping
  archived accounts, accounts with no linked Odoo journal, and
  `odooSourceOfTruth` accounts (which CHB must not push into).
- **Monerium orders refresh on every account pull.** Monerium orders carry the
  human memo and arrive independently of the on-chain mint/redeem, so an
  explicit `chb accounts pull` now always re-fetches them for EURe accounts.
  This fixes a redeem pushing as a bare "burn EURe" because the order (and its
  description) hadn't been pulled yet. Bulk `chb sync` stays gated on a
  blockchain change to avoid needless API calls.

### Odoo journal fix
- **`chb odoo journals <id> fix --since <date>`** (and `fix-amounts --since`)
  scopes the account-reclassification and amount-fix detection to statement
  lines on/after the date (e.g. `--since 20260630`), so you can target recent
  entries without rescanning all history.

### Events
- The events/calendars generate now refreshes `latest/events.json` alongside
  `latest/events.md`. Previously a `chb calendars sync` updated `events.md` but
  left `events.json` stale (frozen at whatever was upcoming the last time
  `chb generate` ran) — so the two could disagree on the number of upcoming
  events. They're now always regenerated together.

## Upgrade Notes

1. To repair a redeem already pushed as "burn EURe": re-pull the account
   (`chb accounts <slug> pull`, now fetches the Monerium order), `chb generate`,
   then `chb odoo journals <id> fix` (rewrites the existing line's description
   from the enriched memo — plain `push` only adds missing lines).
2. To refresh a stale `latest/events.json` immediately, run `chb generate`
   (or `chb generate events`); after this release a `chb calendars sync` keeps
   it in sync automatically.
