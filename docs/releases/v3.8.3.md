# chb v3.8.3

Release date: 2026-07-01

A patch release fixing counterparty-partner resolution for on-chain payouts.

## Highlights

### Odoo
- **Archived partner banks no longer win counterparty resolution.**
  `findOdooPartnerBankByAccountNumber` now filters to `active = true` and routes
  the pick through a small, unit-tested helper (`pickPartnerBankRow`). A stale
  `res.partner.bank` left on the wrong partner — e.g. an employee's IBAN once
  attached to a supplier and later archived — can no longer be matched over the
  live, correct record. When the same IBAN still resolves to more than one active
  partner, a warning is logged: an IBAN should belong to a single partner, and
  silently picking the lowest id has mis-tagged EURe payouts before (salary
  redeems to an employee stamped with a grocery supplier as the partner).

## Upgrade Notes

1. If some historical statement lines are already tagged with the wrong partner
   (because the bad bank record was matched when they were pushed), this release
   only prevents the mis-match going forward — it does not rewrite existing
   lines. Re-point those lines and de-duplicate the offending `res.partner.bank`
   records manually.

## Verification

- `TestPickPartnerBankRow` covers the empty, single, duplicate, orphan, and
  all-orphan cases; full `go test ./...` passes.
- The release build workflow publishes Linux `amd64` and `arm64` assets plus
  `checksums.txt` when the `v3.8.3` tag is pushed.
