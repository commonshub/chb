# chb v3.8.0

Release date: 2026-06-30

A finance-workflow release centered on Odoo reconciliation, per-database cache
isolation, account/journal alignment checks, and safer categorization. It fixes
several cases where already-settled invoices/bills kept showing as open, makes
the invoice/bill reconcile flow clearer and partial-payment aware, and ensures
`fix` never reverts manual work.

## Highlights

### Reconcile · invoices · bills
- Correct "already reconciled" semantics: the `-i` reconcile only warns when a
  bank line is matched to **another invoice/bill**, not when it is merely
  categorized to a GL account (the normal input to reconciliation).
- When a line *is* matched elsewhere, the prompt and the error show the
  conflicting move with clickable Odoo URLs, and offer to detach + re-attach.
- The reconcile review shows a direct Odoo URL to the invoice/bill being
  processed, and links the matched invoice/bill when a line is already attached.
- Partial payments: after attaching a bank line that doesn't cover the full
  amount, the same invoice/bill is re-shown with the **remaining balance** so
  several transactions can be linked until it's fully paid. Re-runs now also
  review **partially-paid** moves (matched against their residual).
- `chb odoo bills|invoices reconcile` now **pulls fresh, then reconciles** (the
  `reconcile` subcommand was previously swallowed by the sync path).
- Reconcile warns when the local cache is stale or — importantly — when a
  **different database** was pulled more recently than the one being reconciled.
- Cache-write fix: a move whose `payment_state` flipped (`not_paid → paid`)
  without a `write_date` bump is now actually rewritten, so paid invoices/bills
  stop reappearing as open after a pull.

### Accounts · per-database namespacing
- Odoo caches are scoped per database under `providers/odoo/<db>/…`, with a
  legacy-path fallback so existing non-namespaced data keeps working until a
  re-pull populates the per-database cache.
- New alignment report in `chb accounts`: flags accounts whose latest local
  balance disagrees with their Odoo **journal** and **GL account** (cached by
  default; `--live` adds the GL-account leg).
- The KBC account is linked to its Odoo journal (28).

### Categorization · journal fix
- `612011 → 612300` across custom and default mappings.
- `cp-order-*` → `drinks` (account `700003`), added to defaults.
- `chb odoo journals <id> fix` categorizes `odooSourceOfTruth` (KBC) lines from
  the **Odoo line narration** via the rules, and is **suspense-only by default**
  — it assigns accounts to still-uncategorized (499000) lines and never reverts
  a manual assignment or an invoice/bill reconciliation. Pass `--remap` to also
  re-account already-assigned history (e.g. to roll out a mapping change).
- Provider-aware narration categorization (e.g. Proximus/Electrabel) on KBC.

### Transactions
- `--amount` is repeatable and ANDed against absolute magnitude, e.g.
  `--amount ">710" --amount "<=800"` keeps `710 < |amount| <= 800`.

### Also included
- In-progress Wise provider scaffolding and a chart-of-accounts GL-code refactor.

## Breaking Changes

- Odoo cache layout is now per-database (`providers/odoo/<db>/…`). Readers fall
  back to the legacy non-namespaced path, so nothing breaks immediately, but
  fresh pulls write to the namespaced location. Keep `ODOO_DATABASE` (or
  `ODOO_URL`) consistent between `pull` and `reconcile`/`fix` so you operate on
  the same database's cache.

## Upgrade Notes

1. Re-pull so the per-database caches and the new `amountResidual` /
   payment-state fields are populated:
   `ODOO_DATABASE=<db> chb bills pull --history` and
   `ODOO_DATABASE=<db> chb invoices pull --history`.
2. `chb accounts` (or `--details` / `--live`) now reports journal/GL alignment;
   investigate any flagged account.
3. To categorize new Odoo-sourced KBC entries (e.g. `cp-order-…` → 700003),
   preview then apply: `chb odoo journals 28 fix --dry-run` then `--yes`.
   Use `chb odoo journals 28 fix --remap` only to re-account already-assigned
   history.
