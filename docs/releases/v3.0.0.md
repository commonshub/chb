# chb v3.0.0

Release date: 2026-05-11

This major release finishes the provider-data refactor and tightens the finance
sync workflow around account inspection, source caches, Odoo journals, and
public settings defaults.

## Highlights

- Adds a compact `chb accounts <account>` detail view with the account name,
  chain/address, an explicit inspection URL, live/computed balance,
  transaction range, sync timestamps, and Odoo journal status.
- Uses Safe only for accounts explicitly marked with `walletType: "safe"`.
  Other blockchain accounts now link to `https://txinfo.xyz/:chain/address/:address`.
- Reports account balance mismatches after the account summary with concrete
  sync/refresh commands.
- Tracks account last sync and last full/history sync separately, including
  JSON output via `lastFullSyncAt`.
- Adds account source/on-chain verification and missing-transfer diagnostics
  after account syncs.
- Moves Odoo journal display names out of `accounts.json` and into a local
  cache to avoid leaking bank identifiers in public defaults.
- Adds default settings files under `cmd/defaults/` and improves setup/settings
  handling for public repo installs.
- Improves Odoo journal sync visibility, duplicate checks, statement fixes, and
  Stripe chronological sync reporting.
- Adds generated source helpers and tests for source/provider outputs.

## Breaking Changes

- Blockchain accounts are no longer assumed to be Safe wallets. Set
  `walletType: "safe"` explicitly when a wallet should link to Safe.
- `accounts.json` no longer stores `odooJournalName`; display names are cached
  locally through `OdooJournalName`.
- Default settings and account config loading now prefer the split settings
  files under `APP_DATA_DIR/settings/`.

## Upgrade Notes

1. Run `chb setup` if local settings files are missing.
2. For Safe wallets, ensure each account has `walletType: "safe"`.
3. Run `chb accounts <account> sync --history` for accounts that have never
   had a full account sync.
4. Run `chb accounts <account> --refresh` after syncing to refresh live balances.

## Verification

- Focused account detail and sync-state tests pass.
- The release build workflow publishes Linux `amd64` and `arm64` assets plus
  `checksums.txt` when the `v3.0.0` tag is pushed.
